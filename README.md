## 🚀 第一阶段：启动与初始化 (激活引擎)

### 0. 启动智能体（仿写驱动）
在对话框中直接发送：
> **"请启动 `novel-writer` 智能体，我要开始写小说了！"**

**【重要】智能体会先索要原著链接，而不是问类型/题材！**
> **"请提供2本以上的原著小说链接，我将分析后仿写。"**

**【内部机制】**：AI 进入仿写驱动模式，流程为：`索要原著链接 → 分析原著 → 初始化叙事状态机 → 仿写正文`

### 1. 提供原著链接
提供 **2本以上** 的原著小说链接，例如：
> http://www.daishuzw.com/daishu/108811.html
> http://www.daishuzw.com/daishu/108837.html

系统会自动爬取并分析原著，提取：类型/题材/世界观/人物原型/剧情模板/节奏模式/爽点结构。

### 2. 系统自动初始化（基于原著分析结果）
分析完成后，系统会**自动推导**以下内容（无需手动输入）：
- **世界观** → `world_rules.md`（从原著提取世界规则框架）
- **人物档案** → `characters.json`（基于原著人物原型，调整"错误信念/内心恐惧"）
- **剧情图谱** → `plot_graph.json`（基于原著剧情模板，Beat化拆解）

系统会展示推导结果供确认，你可以调整后再生成。

## ✍️ 第二阶段：正式写作 (Drafting Loop)

**这是 NovelEngine 最核心的竞争力：Beat 驱动写作。**

### 1. 正文生成指令
**不要说"写第一章"，要说：**
> "现在请根据 `plot_graph.json` 中的【第一个 Beat】开始创作正文。
> **具体要求：**
> - **目标情绪**：重点表现[例如：主角的绝望与不甘]。
> - **字数目标**：[例如：2500] 字。
> - **执行约束**：严格遵守【反总结协议】和【显示而非讲述】原则。"

### 2. 自动化审计流程 (后台运行)
当你发送上述指令后，引擎会自动执行以下**闭环审计**：
`提取 Beat` $\rightarrow$ `生成草稿` $\rightarrow$ `NovelValidator 审计` $\rightarrow$ `(若违规) 自动触发重写` $\rightarrow$ `交付正文`。

**它会自动拦截以下问题：**
- ❌ **总结性结尾**：如果结尾出现了"从此以后..."、"这意味着..."，审计员会强制删掉并要求 AI 重写。
- ❌ **字数注水/不足**：如果字数偏离目标 20% 以上，会自动触发扩写或压缩。
- ❌ **设定冲突**：如果正文违背了 `world_rules.md`，会被标记为异常。

## ✨ 拟人化写作规则 (HUMAN-LIKE WRITING RULES)

**这是消除 AI 味道的关键规则，写正文时必须遵守：**

1. **禁止星号标记内心独白** —— 这是 AI 写作的头号标志。正确做法：
   - 直接融入叙述：`他心想，快到了。`
   - 使用破折号：`——快到了。`
   - ❌ `*他成功了。*` ✅ `他成功了。`

2. **允许不完整言语** —— 真人说话会：
   - 话说到一半：`"算了，其实也没那么..."`
   - 自我打断：`"我不是那个意思，我是说——"`
   - 沉默不语：她转过头。他没说话。

3. **形容词可疑** —— `密道幽深如喉` 太做作，不如 `密道很黑`。

4. **让情绪扭曲** —— 不要直说"她很伤心"，用动作暗示：
   - 她说"随你便"，却已经把指甲掐进掌心。
   - 她笑了。听起来不对劲。

5. **节奏应有混乱感** —— 不是每场戏都流畅：
   - 跳过时间：`三天后——`
   - 突然切换：上一句还在说话，下一句 `月亮升起来了`

6. **具体的笨拙 > 抽象的完美** ——
   - ❌ `她眼中刺痛一闪而逝` ✅ `她揉了揉眼睛，扭过头`
   - ❌ `声音平静却带着倔强` ✅ `她把晶核收进怀里，没接话`

7. **对话应感觉真实** —— 加入口头禅、打断、回避：
   - `"嗯...其实"` `"那个...算了"` `"你先听我说——"` `"别问这个了"`

## ⚠️ 核心禁令 (绝对不要做的事)

| ❌ 错误操作 | ✅ 正确操作 | 理由 |
| :--- | :--- | :--- |
| 直接让 AI "自由发挥"写一章 | 必须指定某个 `Beat` 写入 | 自由发挥会导致 AI 进入"总结模式"，丧失细节。 |
| 在正文中直接修改设定 | 先用 `CharTool` 或 `PlotTool` 修改数据库 $\rightarrow$ 再写正文 | 确保 AI 的长期记忆一致，防止出现前后矛盾。 |
| 接受带有"总结性结尾"的段落 | 要求 AI 重新执行【反总结协议】 | 总结性结尾会破坏小说氛围，使其变成"作文"。 |

## 📂 你的"故事保险箱"在哪里？
所有数据保存在你项目根目录的 `.novelengine/` 文件夹中，请妥善保管：
- `world_rules.md` $\rightarrow$ 世界观法律。
- `characters.json` $\rightarrow$ 人物心理档案（信念/恐惧/弧光）。
- `plot_graph.json` $\rightarrow$ 剧情骨架图（Beat/冲突/伏笔）。
- `outline.md` $\rightarrow$ 小说大纲。
- `chapters/` $\rightarrow$ 章节正文存档（每章独立文件）。
- `novel_config.json` $\rightarrow$ 创作参数配置（总章节数/卷名/字数/Beat数等）。
- `reference_books.json` $\rightarrow$ 原著模板库（由原著参考系统生成）。
- `optimization_guide.md` $\rightarrow$ 创作流程指南（由原著参考系统生成）。
- `plot_templates.json` $\rightarrow$ 剧情模式模板（医术展示/极品亲戚/鉴宝捡漏等）。

**只要这个文件夹在，无论你开启多少个新窗口，AI 都能瞬间找回所有记忆，绝不写崩！**

## 🧹 AI痕迹消除器 (ai_trace_remover.py)

**保存章节时自动运行，也可手动独立调用。**

### 自动消痕
当 `NarrativeVault.saveChapter()` 保存章节时，会自动调用 `ai_trace_remover.py` 对正文进行消痕处理，无需手动操作。

### 手动消痕
```bash
# 默认处理 .novelengine/chapters/ 目录下所有章节
python ai_trace_remover.py

# 处理单个文件
python ai_trace_remover.py <文件路径>

# 处理整个目录
python ai_trace_remover.py <目录路径>
```

### 消痕能力
- **AI腔调词汇替换**：`显著→明显`、`毋庸置疑→没得说`、`恍然大悟→一下明白了` 等 50+ 组
- **对话标签简化**：`沉声道→说`、`冷声道→说`、`颤声道→说` 等 100+ 种
- **长句拆分**：超过 40 字的句子自动拆分
- **口语句注入**：4% 概率注入 `大概/其实/话说回来` 等口语词
- **省略号替换**：1.5% 概率将句号替换为省略号，增加口语感
- **短段落合并**：25% 概率合并相邻短段落，打破 AI 的均匀节奏
- **标点清理**：重复标点、错误破折号等

## ⚙️ 小说配置工具 (novel_config_tool.py)

**交互式或命令行修改创作参数。**

### 交互式使用
```bash
python novel_config_tool.py
```
会显示当前配置并提供菜单选择修改项。

### 命令行快速设置
```bash
# 查看当前配置
python novel_config_tool.py --show

# 快速修改参数
python novel_config_tool.py wordsPerChapter=3000 beatsPerChapter=4
```

### 可配置参数
| 参数 | 说明 | 默认值 |
| :--- | :--- | :--- |
| `totalChapters` | 总章节数 | 235 |
| `chaptersPerVolume` | 每卷章节数 | 50,70,115 |
| `volumeNames` | 卷名 | 断亲逆袭,多领域开花,登顶与真爱 |
| `wordsPerChapter` | 每章字数 | 3000 |
| `beatsPerChapter` | 每章Beat数 | 4 |
| `skillUnlockRate` | 每N章解锁新技能 | 10 |
| `favorabilityGainPerChapter` | 每阶段好感度增量 | 10 |
| `bigFaceSlapRate` | 每N章大打脸 | 5 |
| `smallFaceSlapPerChapter` | 每章小打脸次数 | 1 |

## 📖 原著参考系统

**从两本原著提取核心元素、生成剧情模板库，为创作提供结构化参考。**

### 初始化（首次使用）
```bash
# 命令行直接传入两本原著网址
node init-reference.js http://www.daishuzw.com/daishu/108811.html http://www.daishuzw.com/daishu/108837.html

# 或交互式输入
node init-reference.js
```

### 生成的文件
- `reference_books.json` — 原著模板库（元数据/核心元素/剧情模板/技能解锁/感情规划）
- `optimization_guide.md` — 创作流程指南（原著对比/融合创新/质量控制标准）
- `plot_templates.json` — 剧情模式模板（医术展示/极品亲戚/鉴宝捡漏/感情进展）

### 剧情模板示例
**医术展示**：遇疑难杂症 → 众人质疑 → 主角出手 → 一针见效震惊
**极品亲戚**：亲戚上门 → 无理要求 → 撒泼打滚 → 主角反击
**鉴宝捡漏**：进古玩市场 → 发现真品 → 众人嘲讽 → 鉴定证实

### 质量控制检查清单
每章写完后检查：
- [ ] 完成了 4 个 Beat
- [ ] 至少 1 次打脸情节
- [ ] 好感度推进合理（单章≤10点）
- [ ] 角色行为符合设定
- [ ] 已更新 `plot_graph.json` 和 `characters.json`
